Python Web 框架对比培训
========================

本目录包含了四个主流 Python Web 框架的最小化实现示例，以便进行直观对比。

1. Flask (01_flask)
-------------------
- 类型：微框架 (Microframework)
- 特点：轻量、灵活、简单。它只提供核心功能，其他功能（如数据库、认证）通过插件实现。
- 适用场景：小型应用、原型开发、对灵活性要求高的项目。

2. Django (02_django)
---------------------
- 类型：全栈框架 (Full-stack Framework)
- 特点："内置电池" (Batteries-included)。自带强大的 ORM、管理后台、用户认证、表单处理等。
- 适用场景：中大型应用、企业级项目、需要快速搭建后台的项目。

3. FastAPI (03_fastapi)
-----------------------
- 类型：现代 API 框架
- 特点：高性能（接近 Node.js 和 Go）、基于 Python 类型提示、自动生成 Swagger 文档、原生支持异步 (async/await)。
- 适用场景：高性能 API 服务、微服务、需要强类型验证的项目。

4. Sanic (04_sanic)
-------------------
- 类型：异步框架
- 特点：专注于速度。API 风格类似 Flask，但底层是异步的。
- 适用场景：高并发、高性能需求的 Web 服务。

总结对比表：
| 特性 | Flask | Django | FastAPI | Sanic |
| :--- | :--- | :--- | :--- | :--- |
| 学习曲线 | 低 | 高 | 中 | 中 |
| 灵活性 | 极高 | 低 | 高 | 高 |
| 开发速度 | 快 | 极快 (对于复杂功能) | 快 | 快 |
| 性能 | 中 | 中 | 极高 | 极高 |
| 异步支持 | 较弱 | 部分支持 | 原生支持 | 原生支持 |
